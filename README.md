# MATHS_INSTRUMENTS

> **Концепт-документ** — набор математических инструментов: библиотека формул, калькуляторы, численные методы. Масштаб: личный / учебный.

---

## Что это за проект

Лёгкий веб-инструментарий для математики. Никакого игрового движка (Phaser и подобное — не нужны). Минимальный HTML на фронте, вся математическая логика — на бэкенде или прямо в браузере, в зависимости от модуля.

Основные идеи:
- **Библиотека формул** — просмотр, поиск и отображение формул в LaTeX через MathJax/KaTeX
- **Калькуляторы** — вводишь значения, получаешь результат; от простого (квадратное уравнение) до сложного (матрицы, численное интегрирование)
- **История и хранение** — сохранять прошлые вычисления и свои формулы в базе данных
- **API-first** — фронт остаётся тонким; вся математика живёт на бэке

## Выбор бэкенда: что взять?

### Вариант А — Python (FastAPI)

**Подходит, если** нужна настоящая символьная математика.

```
Стек:
  FastAPI       ← лёгкий асинхронный HTTP-фреймворк
  sympy         ← символьная математика (алгебра, производные, интегралы)
  numpy / scipy ← численные вычисления (матрицы, статистика)
  SQLite        ← хранение формул и истории (файловая БД, нет настроек)
```

Плюсы:
- `sympy` умеет **брать производные, интегрировать, упрощать и решать** уравнения символьно — в JS такого нет
- `numpy` — стандарт для тяжёлых числовых вычислений
- FastAPI автоматически генерирует документацию по API (`/docs`)
- Легко добавить графики позже (matplotlib → PNG-эндпоинт)

Минусы:
- Отдельная среда от фронта (Python ≠ JS)
- Нужно держать сервер запущенным локально или задеплоить

Пример эндпоинта:
```python
@app.post("/решить/квадратное")
def solve_quadratic(a: float, b: float, c: float):
    x = symbols('x')
    return {"корни": [str(r) for r in solve(a*x**2 + b*x + c, x)]}
```

### Вариант Б — TypeScript / Node.js (Hono или Express)

**Подходит, если** хочешь один язык на всём стеке.

```
Стек:
  Hono / Express   ← быстрый HTTP-сервер
  mathjs           ← парсинг и вычисление выражений, матрицы, единицы измерений
  better-sqlite3   ← SQLite для Node
  zod              ← валидация входных данных
```

Плюсы:
- Один язык везде (TypeScript на фронте и бэке)
- `mathjs` умеет вычислять строки вроде `"sqrt(3^2 + 4^2)"` или `"2x^2 + 3x - 5"` напрямую
- Удобно для **вычислителя выражений** (пользователь вводит строку → получает результат)
- Проще задеплоить как единый проект

Минусы:
- Нет символьной алгебры (нельзя взять производную или упростить, как в sympy)
- Нужно аккуратно следить за точностью вычислений

Пример эндпоинта:
```typescript
app.post('/вычислить', async (c) => {
  const { выражение, переменные } = await c.req.json()
  const результат = math.evaluate(выражение, переменные ?? {})
  return c.json({ результат: String(результат) })
})
```

## Рекомендация

| Задача | Выбор |
|--------|-------|
| Символьная математика (производные, интегралы, решение уравнений) | **Python + FastAPI** |
| Вычислитель выражений + единицы + матрицы + один язык | **TypeScript + Node** |
| И то и другое в перспективе | Python-бэк, TS-фронт — два отдельных модуля |

Для **личного / учебного проекта**: бери **Python + FastAPI**, если планируешь символьную математику. Бери **TypeScript**, если хочешь простоту и уже знаешь JS.

## Фронтенд (минимальный HTML)

Никакого фреймворка. Фронт — это просто:
- `<input>` для ввода выражений
- `<div>` для отображения результата (KaTeX для LaTeX)
- Чистый TypeScript, собирается через **Vite**, тесты через **Vitest**

```html
<input id="выражение" placeholder="например: 2x + 3 = 11" />
<button onclick="вычислить()">Считать</button>
<div id="результат"></div>
<script type="module" src="/app.ts"></script>
```

Сборка и запуск:
```bash
npm run dev      # dev-сервер с горячей перезагрузкой
npm run build    # собирает в dist/
npm run test     # запускает Vitest
```

Минимальный `vite.config.ts`:
```typescript
import { defineConfig } from 'vite'

export default defineConfig({
  test: {
    environment: 'node'
  }
})
```
